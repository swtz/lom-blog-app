Aula: 151. Nossas rotas + Entenda Static, Dynamic, SSG e ISR

SSG -> Com o uso desse recurso do Next, o dev informa que o servidor deve
pré-renderizar as rotas estáticas da aplicação.
Essa tecnologia deve ser usado com cuidado, pois o Next armazena essas rotas estáticas
em 'cache', assim, caso o dev não informe que elas devem atualizar (seja por qual
motivo), elas não refletirão nenhuma alteração mesmo que o uso da aplicação modifique
alguma coisa nessa rota.

Assim, o Next entende que o dev quer gerar uma página estática que não se comunica com
seu servidor, ou seja, quando o usuário acessa essa rota estática, o Next busca seu conteúdo
do 'cache' e não do servidor Next.

O Next permite a combinação de rotas estáticas e rotas dinâmicas com o uso da tecnologia
ISR (Incremental Static Regeneration).

Essa tecnologia possibilita que um usuário, após determinado tempo, provoque uma REVALIDAÇÃO
no 'cache' dessa página estática.
Como assim?
Ora, quando o desenvolvedor informa que uma rota será estática, o Next renderiza o conteúdo
presente em seu 'cache' que é gerado no momento da 'build' do projeto.
Com o uso dessa tecnologia, quando essa REVALIDAÇÃO é provocada, o Next REGENERA esse 'cache'
buscando o conteúdo dessa página estática NO SERVIDOR, assim buscando as devidas atualizações,
por exemplo na base de dados. Dessa forma, os próximos usuários do site utilizarão o 'cache'
que fora REGENERADO pelo servidor Next. Esse é o comportamento que o ISG provoca na aplicação.

O Next permite usar ISR, após algum conteúdo do site for alterado, ou seja, não necessariamente por tempo.

Obs.: é preciso distinguir as tecnologias de componentes das tecnologias de rotas que o Next proporciona:

Server Side Rendering ->   componente
Client Side Rendering ->   componente
Static Site Generation ->       rotas
Dynamic Routes --------->       rotas
Incremental Site Regeneration -> rotas

Mapeamento de rotas do projeto The Blog:

/ (Pública) | ISR
/post/[slug] (Pública) | ISR

/admin/post (Privado) | Dynamic | Ler (R) Lista de posts / (D) Delete
/admin/post/[id] (Privado) | Dynamic | Atualizar um post (U)
/admin/post/new (Privado) | Dynamic | Criar um post (C)

/admin/login (Pública) | Dynamic | Fazer o login do usuário
