Aula: 160. Usando unstable_cache para cache de funções (Data Cache)

Aula referência!

Fluxo (abstraído):

Componente dinâmico com 'force-static' -> função em 'cache' -> atualização no bd -> server-action
-> Componente dinâmico usa função com 'cache' atualizado.

Esse comportamento reduz drasticamente os acessos ao banco de dados, pois, enquanto a 'server-action'
não for disparada, o Next busca os dados retornados pela função, que acessa o banco de dados do 'cache'.

Por isso, é bem importante que o desenvolvedor defina as condições que INVALIDAM o 'cache' de modo sábio.

Lembrando: o funcionamento do 'cache' é posto à prova após a 'build' do projeto e com 'npm start'!
