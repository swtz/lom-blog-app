Aula: 140. Cuidado para não mudar um Server Component em Client Component sem querer

Importante: ao usar a diretiva 'use client' em um componente, TODOS OS SEUS FILHOS se tornarão
componentes 'client-side'. Por isso é uma boa prática "empurrar" um componente 'client-side' para
um nível mais baixo no aninhamento de rotas/componentes.

Por conta da maneira que o Server Component é importado, ele se torna um Client Component
de maneira automática. Por isso é muito perigoso usar um componente 'server-side' dentro de um
componente 'client-side'.

Ou seja, todo componente importado para um component 'client-side', torna-se 'client-side'. Isso é
perigoso, pois um componente 'server-side' pode conter informações sensíveis da aplicação.

Como posso resolver esse problema?
Por passar o componente 'server-side' como 'children' do componente 'client-side'.
Assim, cada um se comporta como fora definido:

<ClientComponent>
  <ServerComponent />
</ClientComponent>
