Aula: 166. Movendo o botão para DELETE do post para um Client Component (use client)

Qualquer interação — seja do usuário ou de um sistema — que envolva comunicação com
a base de dados, ao desenvolver com Next, é preciso usar as 'server-actions'.

O funcionamento da 'server-action' usando um elemento JSX 'form' é permitido até
mesmo com o JavaScript do browser desabilitado.

Nessa aula, o coach-Luiz ressaltou que um componente que depende de um 'feedback' do usuário,
deve ser movido para um componente 'client-side' por poder usufruir de mais recursos oriundos
da API do browser.

Qual o objetivo dessa mudança?
Somente o componente que precisa desse 'feedback' será renderizado pelo lado do cliente. Todo
o componente restante que usa ele, pode usar os recursos 'server-side' do Next.
