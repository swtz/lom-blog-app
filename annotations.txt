Aula: 136. Configurando o plugin Tailwind Typography do Tailwind CSS

Ler a documentação do TW sobre '@layer'.

npm i -D @tailwindcss/typography

Esse 'plugin' facilitará o estilização do 'markdown' que fora renderizado.
É uma devDependency, pois, na build final do projeto, ela não será mais necessária,
por conta do processo de transpilação que é feito no código CSS.

Para usar ess 'plugin', é preciso informá-lo no arquivo 'globals.css':

@plugin "@tailwindcss/typography";

É interessante ler mais sobre ele na página do GitHub, pois, a partir de algumas
classes específicas, é que esses estilos serão aplicados nos elementos HTML que
foram renderizados.

Essas classes podem ser informadas nas 'tags' que envolvem o componente React
'ReactMarkdown'.

Dica CSS:

Caso um elemento (table, pre, etc) estoure o container empurrando o documento
inteiro para o lado, é possível envolver esse elemento em uma 'div' e aplicar
a propriamente CSS 'overflow-hidden' no documento. Assim, o dev pode
aplicar a propriedade 'overflow-x: auto', por exemplo, nessa 'div', evitando
que todo o documento seja empurrado para o lado.

Para substituir o conteúdo desse elemento, é usado o componente React 'ReactMarkdown'.
